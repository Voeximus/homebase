Homebase · Health — design direction

From functional to refined

A calm, data-first treatment: one accent used with discipline, semantic macro colors as quiet signals, a tight modular type scale, and editorial breathing room. The goal is Oura/Apple-Health restraint — not gym-bro intensity — while every existing behavior (real-time macros, unit entry, meal containers, Just-me/Together) stays exactly as it is.

01 · Overall direction

Three pillars

Each fixes a specific failure mode in the current screens.

◐

One accent, on purpose

Rose appears only on the primary action and the single most important live number. Macros keep their own hues but as dots + values, never competing fills.

≣

One progress system

The macro card runs three today (ring + calorie bar + three mini-bars). Collapse to a single ring plus one thin track per macro. Less ink, clearer read.

⊞

Editorial rhythm

An 8-pt grid, consistent 20px card padding, and a 6-step type scale replace the ad-hoc sizes and cramped rows.

08 · Adherence & Weight cards

Same language as everything else

These read as cousins, not siblings, today. Align them to the shared scale + accent discipline.

  • Same eyebrow (Label, #8b97a6), same primary number (Stat-L, tabular) — adherence % and weekly-avg should be visually identical stat objects.
  • Adherence: lay streak + compliance as two aligned stats; make the 14-day strip cells a touch taller (h-6) with 3px radius and a hair of gap; keep green/amber/over semantic.
  • Weight: the trend number is the hero (Stat-L, green when on-band); the band track is neutral, the marker bone, the target band a quiet rose tint — same accent rule as the ring.
  • Both cards: 20px padding, the same hairline border, no per-card accent colors beyond the macro/semantic set.
09 · Prioritized

What to do, in order

Quick wins are a single styling pass each; the significant items are the real quality lever.

Quick wins · hours
  • Contrast pass — raise the secondary/label text floors. Biggest read win, cheapest change.
  • Kill the redundant calorie bar on the macro card — the ring already says it.
  • Standardize padding (20), inter-card gap (14), radii (24/18/14), icon sizes (16/18/20).
  • Breathe the meals list — row padding 8→12, meal gap 12, hairline separators.
  • Lighten per-meal "Add food" from filled cyan slab to a quiet text button.
  • Trash treatment — fold delete into a per-meal ⋯ overflow instead of many loud icons.
Significant · the quality lever
  • Restructure the macro card — one ring + three calm rows; remove the dark mini-bar inset.
  • Tokenize palette + type scale as shared CSS classes — the root cause of "inconsistent".
  • Unify toggles/segmented controls to the rose accent (drop the cyan), app-wide.
  • Adherence & weight pass onto the shared stat objects + spacing.

Homebase · Health redesign direction — preserves all current behavior (real-time macros, unit/gram entry, meal containers, Just-me / Together, adherence, auto weight averages). Hex + token values are starting points, tuned to the existing app constants.
